## What Agency Means in Practice

Agency is the ability to choose and carry out actions in service of a goal.

For software, that means the system does more than generate text. It can:
- interpret a task,
- decide on a sequence of steps,
- invoke tools or APIs,
- observe results,
- adjust its plan,
- and continue until the task is done or it needs help.

A chatbot can also do some of these things, but usually only when prompted at each step. The key difference is initiative. A chatbot waits for the next user message. An agent can keep working after the initial request, within the bounds it has been given.

This is why “agency” is not a binary label. It is a spectrum.

## Tool Use Is Not the Same as Being an Agent

A lot of products call themselves agents because they can use tools. But tool use alone does not make a system agentic.

For example, an LLM connected to a search tool can retrieve information. That is useful, but it is still just a model plus a tool call. If the user must ask every next question, approve every step, and manually stitch the result together, the product is closer to a chatbot with extensions than to an agent.

Real tool use in an agentic system has three traits:
1. **Selection** — the system chooses which tool to use.
2. **Sequencing** — the system decides the order of operations.
3. **Recovery** — the system reacts when a tool fails or returns unexpected output.

Frameworks like LangChain have made this easier to build, and model surfaces such as OpenAI’s Responses API support multi-step tool use. But the presence of tools does not settle the question. The important issue is whether the system can use them to advance a goal without requiring constant human steering.

## Goal Persistence: The Feature Most People Miss

Goal persistence is what separates a clever assistant from something that behaves like an agent.

A chatbot is usually optimized for a single turn or a short exchange. It may remember context within the conversation, but it does not typically maintain a durable objective beyond the current interaction.

An agent, by contrast, is built around an objective that survives across steps. If the task is “find the best flight, compare options, and book the one I approve,” the system should not forget the comparison criteria halfway through. It should retain the goal, track progress, and know what still needs to happen.

This is where product design gets hard. Goal persistence can be useful, but it also increases the consequences of mistakes. If an assistant misunderstands the objective, it may confidently keep working in the wrong direction. That is one reason many production products keep the “agent” part narrow and supervised.

## Autonomous Execution: Where the Risk Lives

Autonomous execution means the system can take actions without asking the user at every step.

This is the most visible difference between chatbots and agents, and also the most misunderstood. Autonomous does not mean uncontrolled. In a well-designed system, autonomy is bounded by permissions, scopes, budgets, and stop conditions.

A support bot that drafts a refund request is not the same as a system that actually issues the refund. A travel assistant that proposes three itineraries is not the same as one that books and pays for a ticket. The more a product can change the world outside the chat window, the more it needs safeguards.

That is why many real-world products remain partially manual. They may draft, recommend, summarize, and prepare actions, but require confirmation before executing them. This is not a failure. It is often the correct product choice.

## Where Products Sit on the Spectrum Today

Most AI products are somewhere between a chatbot and a true agent.

Here is a practical spectrum:

- **Chatbot**: answers questions, no external actions.
- **Assistant**: helps with drafting, summarizing, and ideation.
- **Copilot**: works inside a user workflow, often with explicit human approval.
- **Workflow assistant**: can call tools and complete bounded tasks.
- **Agent**: can pursue a goal across multiple steps with limited supervision.
- **Autonomous system**: can execute end-to-end actions with minimal human intervention.

Microsoft Copilot is a good example of the middle of this spectrum: powerful, useful, and often embedded in a human workflow, but still generally structured around user oversight. Claude and OpenAI-based products can also sit anywhere on the spectrum depending on how they are wrapped. The model alone does not determine the product category.

That is the contrarian point: **“agent” is not a model capability. It is a product and systems design choice.** You can build a relatively agentic system on top of a modest model if the workflow is constrained and the tools are reliable. You can also build a very chatty product around a strong model that never really acts.

## A Better Way to Evaluate AI Products

Instead of asking, “Is this an agent?” ask four questions:

1. **Can it use tools?**
2. **Can it keep a goal across multiple steps?**
3. **Can it decide what to do next?**
4. **Can it execute actions without immediate human approval?**

If the answer to only the first question is yes, you probably have a chatbot with integrations. If the answer to all four is yes, you are much closer to an agent.

This framework is useful for builders because it clarifies tradeoffs. More agency can improve usefulness, but it also raises costs around reliability, safety, monitoring, and user trust. The best products do not maximize autonomy everywhere. They choose the right level of agency for the job.

## The Bottom Line

A chatbot talks. An agent acts.

The real difference is not the presence of a model, a UI, or even tools. It is whether the system can persist toward a goal and execute steps on its own within defined limits. Most products today are still hybrids: more capable than a classic chatbot, but not fully autonomous agents.

That is not a weakness. It is the current state of the field. The most effective AI products are usually the ones that are explicit about where they sit on the spectrum—and careful about how much agency they actually grant.
